01

The Planning Rubicon

Stop the "Chat Loop." A true agent separates reasoning from execution. The LLM proposes a plan; a rigid runtime commits it. No plan? No action.

02

Neuro-Symbolic Split

The Brain (LLM) suggests; The Body (Code) decides. We never grant the LLM root access. This separation guarantees safety in production.

03

Governance as Code

Safety must be inline, not an afterthought. The Orchestrator acts as the system's "conscience," using OPA Policies to veto dangerous actions before they execute.

04

Strict Contracts

Inputs must be parsed and validated (Pydantic/JSON Schema). If the output doesn't validate, the program crashes intentionally rather than hallucinating a parameter.

05

The OODA Loop

Observe, Orient, Decide, Act. The architecture persists state, logs outcomes, and feeds results back into memory to anticipate failure.

A true agent must produce a committable, inspectable plan. The system requires commitment, grounding, and temporality.
