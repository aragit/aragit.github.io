The Neuro-Symbolic Taxonomy

We classify implementations using the academic taxonomy established by Kautz, Garcez, and d'Avila Garcez. By varying the structural relationship between neural and symbolic components, we optimize along a strict spectrum of liability. Our website taxonomy maps directly to these academic foundations.

TYPE 3

THE CITADEL Academic Type 2 · Symbolic[Neuro]

Life-critical / zero-tolerance
Stack focus: Dominant Governance (06) and Execution (05)

The symbolic core owns execution. Neural networks are strictly nested inside and subordinate to deterministic guards. In high-stakes environments, an LLM must never directly control execution paths. The neural engine acts as an upstream strategist, parsing complex inputs to propose intent. This intent is compiled into a Directed Acyclic Graph (DAG). Before any node executes, a stateless symbolic validation layer evaluates the plan against hard mathematical and physiological constraints.

TYPE 4

THE NEXUS Academic Type 6 · Neuro[Symbolic]

High-stakes discovery & reasoning
Stack focus: Dominant Reasoning (04) and Meta-Cognition (07)

The neural core drives open-ended exploration. Symbolic verifiers — ontologies, graph constraints, biophysical validators — are embedded directly inside the neural reasoning loop. When problems are too complex for static DAGs, the neural core must lead using dynamic planning cycles (ReAct, LangGraph state machines). To prevent drift, symbolic components are placed inside the model's observation loop, not outside it.

TYPE 5

THE ARBITER Academic Type 5 · Symbolic ↔ Neural

Competitive markets & dynamic environments
Stack focus: Dominant Tool Registry (03) and Governance (06)

Parallel symbolic and neural tracks interact continuously. Neither dominates; a formal mathematical engine arbitrates conflicts to maintain system invariants. When multiple autonomous agents compete, trusting heuristics or unconstrained model outputs introduces unpredictable economic risk. Neural agents maximize individual objectives, but a compiled game-theoretic engine enforces absolute structural boundaries.

AXIOMIS

AXIOMIS is our Type 2 (Symbolic[Neuro]) clinical intelligence system with a deterministic FHIR state machine at its core. We are migrating toward Type 6 (Neuro[Symbolic]) to enable more flexible reasoning in open-ended diagnostic scenarios — but Type 2 safety invariants are non-negotiable:

  • Every treatment plan compiles to a DAG before execution.
  • Every node is pre-flight checked by OPA/Rego policies.
  • Every execution trace is committed to an immutable ledger.
  • Every override of a neural recommendation requires explicit symbolic justification.

This is how we cross the Planning Rubicon without crossing the safety line.
