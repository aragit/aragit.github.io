Architecture of Agentic Systems

The maturity of enterprise AI is measured not in model scale, but in how reliably probabilistic reasoning is bound by deterministic execution.

We do not deploy models or optimize prompts. We engineer compound systems — architectures where neural reasoning and symbolic governance are composed into standardized layers that transfer across high-liability industries.

Architecture Evolution Path

Most production systems begin as Type 2 (The Citadel) with deterministic safety guarantees, then migrate toward Type 6 (The Nexus) as symbolic validation frameworks mature. The 7-layer stack remains constant; only the control flow inverts — from symbolic core with neural advisors to neural core with symbolic verifiers.

This is the trajectory of our own AXIOMIS/SentriXIA platform: currently Type 2 with R2 (FHIR integration, episode state machine), targeting Type 6 at R3 (dynamic DAG, COGITATOR neural core).

TYPE 3

THE CITADEL Academic Type 2 · Symbolic[Neuro]

Life-critical / zero-tolerance

The symbolic core owns execution. Neural networks are strictly nested inside and subordinate to deterministic guards. In high-stakes environments, an LLM must never directly control execution paths. The neural engine acts as an upstream strategist, parsing complex inputs to propose intent. This intent is compiled into a Directed Acyclic Graph (DAG). Before any node executes, a stateless symbolic validation layer evaluates the plan against hard mathematical and physiological constraints.

Failure Modes & Recovery
  • Neural planner deadlock: When no proposed plan passes symbolic validation, the system escalates to a deterministic fallback heuristic (e.g., most conservative safe option) and flags for human review.
  • Guard bypass attempt: All execution paths are compiled to a DAG before runtime. There is no runtime interpreter that can skip validation nodes.
  • Stale policy corpus: Policies are versioned and cryptographically signed. A policy update triggers a full re-validation of all pending plans in the queue.

The neural core drives open-ended exploration. Symbolic verifiers — ontologies, graph constraints, biophysical validators — are embedded directly inside the neural reasoning loop. When problems are too complex for static DAGs, the neural core must lead using dynamic planning cycles (ReAct, LangGraph state machines). To prevent drift, symbolic components are placed inside the model's observation loop, not outside it.

Failure Modes & Recovery
  • Max self-correction exceeded: After 3 regeneration cycles, confidence is floored to 0.0 and control transfers to a Type 2 fallback (static DAG with pre-approved heuristics).
  • Verifier contradiction: When symbolic verifiers disagree, the system invokes a meta-cognitive arbitration layer that weights verifiers by historical accuracy on the current domain.
  • Confidence decay stall: If confidence stops changing between iterations, the system treats this as convergence failure and triggers human handoff.
TYPE 5

THE ARBITER Academic Type 5/9 · Symbolic ↔ Neural

Competitive markets & dynamic environments

Parallel symbolic and neural tracks interact continuously. Neither dominates; a formal mathematical engine arbitrates conflicts to maintain system invariants. When multiple autonomous agents compete, trusting heuristics or unconstrained model outputs introduces unpredictable economic risk. Neural agents maximize individual objectives, but a compiled game-theoretic engine enforces absolute structural boundaries.

Failure Modes & Recovery
  • Nash non-convergence: If equilibrium is not reached within 500 iterations, the system falls back to a correlated equilibrium computed by a deterministic LP solver.
  • Budget invariant violation: Any agent exceeding its budget is immediately suspended. Its allocated resources are redistributed via proportional fairness.
  • Strategy cycle detection: Best-response dynamics are monitored for cycles. If detected, the system switches to fictitious play (time-averaged strategies).

The Research Frontier

For unconstrained scientific discovery, our multi-agent research environments use foundational model coordination with the Model Context Protocol (MCP). The core engine dynamically spins up secure execution containers, pulling literature, running calculations, updating citation graphs, and self-correcting based on formal logic validations.
